SourArts
Gallery

Sourarts adalah platform berbasis web untuk interaksi jual dan beli jasa gambar antara Artist dan Pelanggan.

Identifikasi Potensi Lokal

Banyak seniman lokal memiliki karya berkualitas namun terjebak dalam metode transaksi manual di media sosial yang rentan penipuan dan pencurian karya (screenshot/save image tanpa izin). Website ini hadir sebagai jembatan profesional yang mengotomatisasi transaksi dan melindungi hak cipta.

Klien Puas

Dari sudut pandang Teknik Informatika, fokus utama website ini bukan sekadar menampilkan gambar, melainkan pada keamanan aset digital dan otomasi alur kerja. Dengan platform ini, potensi seni lokal tidak lagi hanya menjadi konsumsi tetangga, melainkan menjadi komoditas ekonomi digital yang terlindungi dan profesional.

Analisis Kebutuhan Sistem

💬

Fungsional

Terdapat Album untuk melihat gambar dari seniman, memiliki password hash di dalam database

🏢

Non-Fungsional

Memiliki tampilan website yang responsive di berbagai layar handpone ataupun dekstop

📄

Role

Terdapat User, Artist dan Admin

Konsep

konsep ini adalah platform galeri digital yang mengotomatisasi jual-beli karya seni lokal secara aman


Arsitekur

sistem bekerja dengan memisahkan file gambar menjadi dua jenis: gambar preview rendah resolusi dengan watermark untuk publik, dan file asli berkualitas tinggi yang disimpan di server tertutup. Transaksi diproses melalui gerbang pembayaran otomatis, di mana pembeli hanya akan mendapatkan akses unduh file asli via tautan khusus setelah pembayaran terverifikasi, sementara Admin bertugas memoderasi konten dan mengelola perputaran dana antara pembeli dan seniman.

Activity

Hubungan Aktor & Use Case Koneksi dan relasi antar elemen dalam sistem

1

INCLUDE

Association adalah hubungan langsung antara aktor dengan use case. Artinya, aktor berinteraksi langsung dengan sistem.

Pelanggan —— Login
Pelanggan —— Buat Pesanan
Artist —— Terima Pesanan

2

ASSOCIATION

Include adalah hubungan dimana suatu proses selalu melibatkan proses lain. Jadi, use case utama tidak bisa berjalan tanpa proses ini.

Buat Pesanan : include Progress Pesanan
Bayar Pesanan : include Proses Pembayaran

3

EXTEND

Extend adalah hubungan tambahan yang bersifat opsional. Use case ini hanya berjalan dalam kondisi tertentu.

Beri Ulasan
extend Progress Pesanan
